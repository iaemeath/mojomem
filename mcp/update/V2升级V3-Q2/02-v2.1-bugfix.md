# 方案 V2.1：修 bug（过渡版本）

> 状态：✅ 采纳（过渡，非架构性改动）
> 时间：2026-07-13

## 核心改动

V2.0 基础上修了三个问题，不涉及架构变化：

### 1. `_delete` 文档与实现不一致

**问题**：schema 注释和工具描述说"软删除（标 deleted_at）"，但 `_delete()` 实际执行 `DELETE FROM` + `VACUUM`（物理删除）。

**修复**：描述对齐代码，改为"彻底硬删除记忆及向量索引，立即生效"。

### 2. VACUUM 时机优化

**问题**：每次 `_delete` 都执行 `VACUUM`，高频删除时全表压缩很慢。

**修复**：VACUUM 从 `_delete` 移到 `_init`（启动时执行一次）。

### 3. 版本号更新

`serverInfo.version`: `"2.0"` → `"2.1"`

## 代码变化

```python
# V2.0
def _delete(self, args):
    conn.execute("DELETE FROM memory_facts WHERE obs_uuid=?", (obs_id,))
    conn.commit()
    conn.execute("VACUUM")  # ← 每次删除都 VACUUM，慢
    conn.close()
    return {"hard_deleted": n}

# V2.1
def _delete(self, args):
    conn.execute("DELETE FROM memory_facts WHERE obs_uuid=?", (obs_id,))
    conn.commit()
    conn.close()  # ← 不再每次 VACUUM
    return {"hard_deleted": n}

def _init(self):
    ...
    conn.execute("VACUUM")  # ← 移到启动时
```
