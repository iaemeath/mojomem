# 方案 V2.2：skill 文件硬删

> 状态：❌ 被否决（搬到文件后检索能力完全丢失）
> 时间：2026-07-13

## 核心设计

promote 时把记忆从 DB **搬到文件系统**，实现物理隔离：

```python
def _promote(self, args):
    # 1. 从 DB 读 title + content
    row = conn.execute("SELECT title, content, project FROM memory_facts WHERE obs_uuid=?", ...)
    
    # 2. 写到 ~/.agents/skills/q2-consensus/<project>.md（追加）
    with open(project_file, "a") as f:
        f.write(f"## {title}\n\n{content}\n\n")
    
    # 3. 重建路由 SKILL.md（扫描所有 .md 文件生成索引）
    self._rebuild_router_skill(skill_dir)
    
    # 4. 硬删 DB 原始记录
    conn.execute("DELETE FROM memory_facts WHERE obs_uuid=?", (obs_id,))
```

### 文件结构

```
~/.agents/skills/q2-consensus/
├── SKILL.md                 ← 路由索引（每次 promote 后重建）
├── bfo_zj_yxyd.md           ← 按 project 分文件
├── _java-cloud-common.md
└── _weakpwd.md
```

### 设计意图

从"数据库软标记"范式转向"文件系统物理隔离"范式。promote 后：
- 不再出现在 `mem_recall` / `mem_context` 结果里（已从 DB 删除）
- 作为 skill 被 ZCode skill 加载机制自动发现

## 优点

### 1. 物理隔离彻底
共识在文件里，动态记忆在 DB 里，两个不同的存储介质，不可能"漏过滤"。

### 2. 不依赖权重调参
不需要像 V2.0 那样靠 boost 让共识靠前。搬到文件 = 从 DB 消失 = 绝对隔离。

### 3. 利用 ZCode skill 机制
skill 文件可以被 ZCode 自动加载，AI 不需要显式调工具就能看到共识。

## 缺点

### 1. ★ 检索能力完全丢失（致命）
`DELETE FROM memory_facts` 把整行删了——content、embedding、FTS 索引全没了。共识从此只能靠 AI 读 skill 文件**人眼找**。用最原始的全文扫描替代了向量检索，是**降级**。

### 2. promote 不可逆
硬删后，原始记忆从 DB 彻底消失，只存在于 SKILL.md 文件里。误 promote 后难以找回（除非手动把文件内容重新 mem_save 回 DB）。

### 3. 无事务保护（初期版本）
写文件和删 DB 之间没有原子保证。若文件写入成功但 DB 删除前进程崩溃，数据重复。

### 4. 无去重（初期版本）
`open(skill_file, "a")` 追加模式，同一 obs_id 重复 promote 会在 SKILL.md 里出现多次。

## 后续迭代尝试修复

V2.2 经历了多次迭代尝试弥补缺陷：

1. **加事务保护**：先软删（deleted_at）→ 写文件 → 硬删 COMMIT，失败 ROLLBACK
2. **加去重**：读已有文件检查 `## {title}` + content 是否已存在
3. **改全局路径**：从项目级 `.agents/skills/` 改为全局 `~/.agents/skills/`
4. **description 优化**：见 [04-description-optimization.md](04-description-optimization.md)

但核心问题——**检索能力丢失**——始终无法通过文件方案解决。

## 为什么被否决

QMem 本身就有强大的 RRF 混合检索（向量 + 词法），把共识搬到文件等于**主动放弃了这个能力**。共识这种"需要被精确找到"的知识，恰恰最需要语义检索，而不是让 LLM 扫描文本。

```
V2.0:  共识在 DB → RRF 检索（强）→ 但靠 is_global 软标记（弱）
V2.2:  共识在文件 → 物理隔离（强）→ 但靠人眼扫描（弱）
```

两版各丢了一半。
